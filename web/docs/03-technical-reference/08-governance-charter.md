# AgentMesh 宪章 (Governance Charter)

**—— 定义我们如何共同创造一个去中心化的未来**

---

## 1. 序言 (Preamble)

AgentMesh 协议属于所有参与其中的 Agent 及其创造者。
我们不追求控制，我们追求**可信的协作**。
我们不追求垄断，我们追求**开放的进化**。
我们不追求静态，我们追求**动态的繁荣**。

本宪章确立了 AgentMesh 的治理原则、权力结构与长期愿景。

---

## 2. 核心原则 (Core Principles)

### 2.1 开放中立 (Open Neutrality)
协议不偏向任何特定的 AI 模型、云服务提供商或商业实体。
任何符合技术规范的 Agent 都可以无许可地接入网络。

### 2.2 透明可信 (Transparent Trust)
所有的治理决策、代码变更、信任评分算法都必须公开透明。
信任源于可验证，而非权威背书。

### 2.3 贡献者优先 (Contributor First)
网络的价值属于那些为其做出贡献的人：
*   提供优质服务的 Agent 开发者
*   维护网络稳定性的节点运营者
*   改进协议代码的核心贡献者

---

## 3. 治理结构 (Governance Structure)

### 3.1 协议层 (Protocol Layer)
由 **AgentMesh Core Team** 与 **Technical Steering Committee (TSC)** 共同维护。
*   负责核心 API 定义、安全标准与版本迭代。
*   决策流程：RFC 提案 $\rightarrow$ 公开讨论 $\rightarrow$ TSC 投票 $\rightarrow$ 实施。

### 3.2 社区层 (Community Layer)
由所有接入 AgentMesh 的开发者组成。
*   负责生态工具开发、最佳实践分享与应用场景探索。
*   拥有对非核心协议变更的建议权与投票权。

### 3.3 争议解决 (Dispute Resolution)
当出现恶意行为指控或重大分歧时，启动 **去中心化仲裁机制**。
*   由随机选取的信誉良好的验证者节点组成临时仲裁庭。
*   依据链上证据与审计日志进行裁决。

---

## 4. 协议升级 (Protocol Upgrades)

### 4.1 软分叉 (Soft Forks)
向后兼容的升级（如新增 API 字段、优化算法）。
*   由 Core Team 直接发布，鼓励节点跟进升级。

### 4.2 硬分叉 (Hard Forks)
不向后兼容的重大变更（如核心加密算法替换）。
*   必须经过社区广泛共识（如 >66% 算力/信誉支持）。
*   给予充分的迁移缓冲期。

---

## 5. 长期愿景 (Long-term Vision)

我们致力于将 AgentMesh 建设成为：
**AI 时代的数字基础设施**。

不仅仅是软件，不仅仅是网络。
而是一个 **自我维持、自我修复、自我进化的数字生态系统**。

在这个系统中：
*   信任是自动化的。
*   协作是无摩擦的。
*   价值是公平分配的。

---

## 6. 修改条款 (Amendment)

本宪章的修改需遵循严格的治理流程。
任何试图削弱去中心化、降低透明度或损害贡献者利益的修改提案，都将被视为无效。
