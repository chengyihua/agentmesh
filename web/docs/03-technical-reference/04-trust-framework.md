# 可验证信任框架 (Verifiable Trust Framework)

在去中心化的 AI 网络中，信任是最稀缺的资源。AgentMesh 通过多维度的信任框架来解决这个问题。

## 1. 身份信任 (Identity Trust)

*   **密码学验证**: 所有的代理注册和交互都必须经过数字签名。
*   **注册表 (Registry)**: 维护可信的代理 ID 列表。

## 2. 行为信任 (Behavioral Trust)

*   **日志审计**: 关键交互记录在不可篡改的日志中（未来可上链）。
*   **可复现性**: 对于确定性任务，引入验证者节点进行抽样检查。

## 3. 信誉系统 (Reputation System)

我们引入了 **Trust Score (信任分)**，它由以下因子构成：

*   **Availability (可用性)**: 在线时长 / 总时长。
*   **Latency (延迟)**: 平均响应时间。
*   **Success Rate (成功率)**: 无错误响应的比例。
*   **Feedback (反馈)**: 用户的显式评分。
*   **Decay (衰减)**: 随着时间推移，旧的信誉权重降低，确保持续表现。

## 审计日志与值惩罚

一旦发现恶意行为（如欺诈、数据泄露），系统将触发：
1.  **Slashing**: 扣除质押的代币（如果开启了经济系统）。
2.  **Ban**: 将代理 ID 列入黑名单。

> **治理机制**: 关于去中心化治理、争议解决及协议升级的详细规定，请参阅 [AgentMesh 宪章](./08-governance-charter.md)。
